**نَوَاة (Nawah)** ليس مجرد تطبيق لإدارة الأعمال؛ بل هو "نظام تشغيل مؤسسي ذكي" ومستقل بالكامل، يعتمد على معمارية الذكاء الاصطناعي التوكيلي (Agentic AI). تم تصميم النظام للقضاء على "الشلل التشغيلي" في المؤسسات الكبرى عبر جيش من الوكلاء الافتراضيين الذين يتواصلون، يحللون، ويتخذون القرارات لتنفيذ المعاملات المعقدة من البداية وحتى التسليم النهائي، بصورة آلية 100% وبدون تدخل بشري في مسار العمل.

---

## 🚀 النظرة العامة (Executive Overview)

تعاني المؤسسات اليوم من بطء شديد وهدر مالي بسبب انعزال الأنظمة البرمجية عن بعضها (Silos) واعتمادها على الموظف البشري لنقل البيانات وإدارة المهام الروتينية. **(نَوَاة)** يحل هذه المشكلة جذرياً عبر:
* استشعار المهام والطلبات من كافة المداخل (بريد إلكتروني، ملفات، واجهة مستخدم).
* توجيه المهام آلياً عبر "مُوجّه تكتيكي" (Dispatcher).
* تنفيذ المعاملات الشاملة (مثل التوظيف، المالية، الشؤون القانونية) عبر سرب من الوكلاء المتخصصين.
* تقديم نتائج نهائية مقطرة وموثوقة للقائد البشري لاعتمادها بضغطة زر (Human-in-the-Loop for approvals only).

## 🏗️ المعمارية الهندسية (Three-Tier Agentic Architecture)

تم بناء المنظومة بهندسة كائنية (OOP) موزعة على ثلاث طبقات متكاملة، تتواصل فيما بينها عبر مساحة عمل مشتركة ومسار بيانات ديناميكي:

### 1. طبقة الاستشعار والاستقبال الذكي (L1 - Omni-Channel Ingress)
تعمل كرادارات يقظة تمتص الطلبات من مصادر متعددة وتقوم بتكوين "وعي شامل" بالمهمة.
* **Email Watcher & Folder Watcher:** استشعار فوري للملفات والرسائل الواردة.
* **Ingress Firewall:** جدار حماية لصد البيانات الخبيثة وتجاهل الرسائل المزعجة (Zero-LLM Filter) لتوفير التكاليف.
* **Task Synthesizer:** محلل النوايا الذي يجبر النماذج اللغوية على إخراج بيانات منظمة ومحمية (JSON Schema via Pydantic) لمنع الهلوسة.

### 2. طبقة التخطيط وتوزيع المهام (L2 - Cognitive Triage Dispatcher)
عقل التخطيط المركزي الذي يستلم الطلبات المنظمة ويقوم بـ:
* تفكيك المهام المعقدة إلى خطوات إجرائية.
* توجيه المهمة (Semantic Routing) وتحديد الوكلاء المطلوبين من الطبقة التنفيذية لضمان أقصر مسار وأعلى دقة.

### 3. سرب الوكلاء التنفيذي (L3 - Executive Agent Swarm)
قلب المنظومة النابض؛ فريق من الخبراء الرقميين يعملون كخلية نحل عبر مساحة ذاكرة مشتركة (`Agent Workspace`):
* **L3A (الوكلاء التحليليون):** مثل `AnalystAgent` و `MathAgent` و `VisionAgent` لقراءة المستندات الطويلة، حل المعادلات، واستخراج البيانات.
* **L3B (الوكلاء التنفيذيون):** مثل `HR Agent` و `Finance Agent` و `Legal Agent` لاتخاذ القرارات المؤسسية والتفاوض وبناء العقود.

## 🛡️ دروع الحماية والحوكمة (19 Active Guardrails & Security)

في (نواة)، الأمن والامتثال ليسا إضافة ثانوية، بل هما في صميم المعمارية. النظام محصن بـ 19 درعاً برمجياً صارماً، أبرزها:

1. **جدار الحماية المالي (Finance Hard-Lock):** يمنع الوكلاء من تجاوز الميزانيات المعتمدة أو تقديم عروض مالية تفوق السقف المحدد.
2. **محرك مكافحة التزوير (Anti-Fraud BGCheck):** يتقصى الحقائق ويقاطع بيانات السير الذاتية مع السجلات الرسمية لاكتشاف الادعاءات الكاذبة فوراً.
3. **فلتر الاختراق الحواري (Prompt Injection Firewall):** يرصد ويحظر أي محاولات تلاعب أو حقن أوامر خبيثة تهدف للسيطرة على الوكلاء.
4. **بوابة الامتثال للتوطين (Nitaqat / Saudization Gate):** تضمن توافق قرارات التوظيف مع قوانين العمل المحلية (مثل نسب التوطين) قبل البدء بأي إجراء.
5. **محوّل النيران الذكي (LLMFailover Router):** نظام موازنة آلي (Load Balancing) ينتقل بسلاسة بين نماذج الذكاء الاصطناعي (Claude / Gemini) في أجزاء من الثانية عند تعطل أحدهم (Rate Limit 429) لضمان استمرارية العمل (100% Uptime).

## 💻 التقنيات المستخدمة (Technology Stack)

* **اللغة الأساسية:** Python 3.11 (لأعلى كفاءة في معالجة النظم الموزعة).
* **بوابة الواجهة الخلفية (Backend API):** `FastAPI` و `Uvicorn` لمعالجة الطلبات غير المتزامنة بسرعة فائقة.
* **الواجهة الأمامية المزدوجة (Hybrid Frontend):**
  * واجهة تفاعلية ديناميكية مبنية بـ `React` / `Vite` للتحكم المتقدم.
  * لوحة قيادة زجاجية (Glassmorphism Dashboard) متصلة بالواجهة الأساسية عبر تكتيك `MutationObserver` المتزامن.
* **إطار عمل الوكلاء والذاكرة:** `LangChain` و `LangGraph` لتمكين التفكير الاستدلالي (ReAct) وإدارة حالة الوكلاء المتعددين.
* **قاعدة البيانات (State Management):** `SQLite` مع تفعيل نمط الكتابة الاستباقية (`WAL - Write-Ahead Logging`) لضمان موثوقية المعاملات.
* **التحقق من البيانات (Validation):** `Pydantic`.

## ⚙️ دليل التثبيت والتشغيل (Quick Start Guide)

لتشغيل منظومة نواة في بيئتك المحلية، يرجى اتباع الخطوات العسكرية التالية:

### 1. إعداد البيئة (Environment Setup)
```bash
# استنساخ المستودع
git clone [https://github.com/Ahmed-MiX/Nawah_Project.git](https://github.com/Ahmed-MiX/Nawah_Project.git)
cd Nawah_Project

# تفعيل البيئة الوهمية
python -m venv venv
source venv/bin/activate  # في بيئة الماك/لينكس
venv\Scripts\activate     # في بيئة الويندوز

# تثبيت الترسانة (المكتبات والمتطلبات)
pip install -r requirements.txt

2. تشغيل عقل المنظومة (Backend Server)

# إطلاق الخادم الأساسي لمعالجة المهام وتفعيل الوكلاء
python start_nawah.py
# أو عبر الخادم المباشر
uvicorn core.api_gateway:app --reload

3. الدخول إلى غرف القيادة (UI Dashboards)
بمجرد عمل الخادم الأساسي (على المنفذ 8000)، يمكنك التوجه إلى الواجهات عبر المتصفح:

الواجهة المؤسسية التفاعلية: http://localhost:8000/

لوحة القيادة المتقدمة (Pipeline Dashboard): http://localhost:8000/dashboard.html
